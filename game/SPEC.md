# 엘레베이터 기다리는 게임 — 개발 스펙

## 개요

| 항목 | 내용 |
|------|------|
| 제목 | 엘레베이터 기다리는게임 |
| 플랫폼 | 단일 HTML 파일 (브라우저 바로 실행) |
| 기술 스택 | 순수 HTML / CSS / JS (외부 라이브러리 없음) |
| 총 플레이 시간 | 고정 60초 |
| 화면 기준 | 데스크탑 1280×720 |

**컨셉:** 회사 10층에서 엘리베이터를 기다리는 일상의 스트레스를 60초 안에 압축 재현.
유저가 호출 버튼을 누를 때마다 다른 방식으로 엘베를 놓치며 분노가 쌓이고,
분노 100 도달 시 엘베를 직접 부수는 **SMASH 모드**로 전환해 카타르시스를 제공.

---

## 화면 레이아웃

```
┌─────────────────────────────────────┐
│ 분노 0/100 [====        ]     10F   │  ← 좌상단 분노 게이지 / 우상단 현재 층
│                                     │
│   [B2]      [9F]      [1F]          │  ← 엘베 3대 (세로 샤프트 안 위/아래 이동)
│   엘베A     엘베B     엘베C          │
│                                     │
│       "엘레베이터를 부르세요"         │  ← 상태 텍스트 (이벤트 결과 문구)
│                                     │
│           [  호출  ]                │  ← 버튼 1개 (강조 애니메이션 적용)
└─────────────────────────────────────┘
```

### 엘베 샤프트
- 3대 각자 현재 층수를 숫자로 표시
- 이벤트 발생 시 해당 엘베가 10F를 향해 움직이는 애니메이션 재생 → 실패 처리

### 분노 게이지 색상 단계

| 범위 | 색상 |
|------|------|
| 0 ~ 30 | 회색 |
| 30 ~ 60 | 코랄 |
| 60 ~ 90 | 레드 |
| 90 ~ 100 | 강렬한 레드 + 흔들림 이펙트 |

```js
phase: 'intro' | 'waiting_click' | 'event_playing' | 'smash' | 'ended'
currentEventIndex: 0 ~ 7
rage: 0 ~ 100
```

### 진행 규칙
- 전체 흐름은 **스크립트된 8개 이벤트** 고정 순서로 진행 (절대 랜덤화 금지)
- 각 이벤트는 **유저가 호출 버튼을 눌러야만** 다음 단계 진행 (자동 진행 없음)
- **60초 마스터 타이머**는 진행 상태와 완전히 독립적으로 흐름
- 60초 도달 시 진행 상태 무관하게 즉시 게임 종료 화면으로 강제 전환

## 이벤트 시나리오 (고정 순서)

| # | 이벤트 설명 | 분노 증가 | 관련 엘베 | 결과 애니메이션 |
|---|------------|----------|----------|----------------|
| 1 | 지하2층 엘베가 잡힘 (가장 먼 엘베 응답) | +8 | A (B2) | A가 B2 → 10F로 서서히 이동 후 그냥 도착 |
| 2 | 도착했지만 사람이 꽉 차서 못 탐 | +14 | A | 문 열림 애니메이션 + "만원" 텍스트 → 다시 닫힘 |
| 3 | 재호출 → "고장" 표시, 취소 | +10 | B (9F) | B가 움직이려다 멈추고 빨간 "고장" 배지 표시 |
| 4 | 다른 엘베가 방향 바꿔 내려감 | +12 | C (1F) | C가 위로 올라오다 갑자기 아래로 방향 전환 |
| 5 | 문 열리려다 다시 닫힘 (반응 늦음) | +10 | A | 문이 살짝 열렸다가 유저 반응 전에 닫힘 |
| 6 | 옆 동료가 새치기해서 탐 | +14 | B | 문 열림 + 작은 캐릭터 아이콘이 먼저 들어가고 문 닫힘 |
| 7 | "점검중" 표시로 멈춤 | +12 | C | C 위에 "점검중" 배지, 움직임 정지 |
| 8 | 도착 직전 다른 층 호출에 뺏김 | +20 | A | A가 9F까지 왔다가 다른 층으로 방향 전환 → 분노 100 도달 |

> 분노 합계: 8+14+10+12+10+14+12+20 = **100**
> 이벤트 8 종료 순간 분노 정확히 100 → 즉시 SMASH 모드 전환

각 이벤트 사이 분노 증가는 **게이지 애니메이션 (숫자 카운트업 + 바 채워짐)** 으로 명확하게 표시.

## 유저 입력 강조 로직

버튼이 활성화된 후 유저가 누르지 않을 경우 시간에 따라 강조 단계적 강화:

| 경과 시간 | 강조 방식 |
|----------|----------|
| 0 ~ 2초 | 기본 펄스 (은은한 scale 애니메이션) |
| 2 ~ 4초 | 펄스 속도 증가 + 버튼 테두리 글로우 추가 |
| 4 ~ 6초 | 화면 살짝 흔들림 + 버튼 크기 커짐 + "빨리 눌러요!" 문구 깜빡임 |
| 6초 이상 | 강조 최대치 유지 (화면 흔들림 + 강한 글로우 + beep 경고음 반복) |

- **절대 자동 진행 없음** — 유저 클릭 시에만 다음 이벤트 진행
- 사운드: Web Audio API로 간단한 beep 합성 (외부 오디오 파일 불필요)

## SMASH 모드

분노 100 도달 후 남은 시간(약 15~16초) 동안 활성화.

- "호출" 버튼 → **"부수기"** 버튼으로 교체
- 엘베 3대를 직접 클릭/탭 가능
- 클릭 시: 파편 튀는 애니메이션 + 화면 흔들림 + 타격음 재생
- 부순 횟수 카운터 표시 (선택 사항)
- 60초 도달 시 스매시 모드 진행 중이어도 즉시 게임 종료 화면 전환

## 기술 구현 참고

- **타이머:** `requestAnimationFrame` 또는 `setInterval`로 60초 마스터 타이머 관리
- **렌더링:** CSS transition/keyframe 위주 권장 (애니메이션 다수)
- **사운드:** Web Audio API로 beep/타격음 합성 (외부 파일 없음)
- **단일 파일:** HTML/CSS/JS 모두 인라인 구성
