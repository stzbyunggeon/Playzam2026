# 엘레베이터 기다리는 게임 — 개발 스펙

## 개요

| 항목 | 내용 |
|------|------|
| 제목 | 엘레베이터 기다리는게임 |
| 플랫폼 | HTML/CSS/JS (브라우저 바로 실행) |
| 기술 스택 | 순수 HTML / CSS / JS (외부 라이브러리 없음) |
| 화면 기준 | 데스크탑 1280px 기준 |
| 파일 구조 | `index.html` / `css/style.css` / `css/game.css` / `js/utils.js` / `js/game.js` / `js/main.js` |

**컨셉:** 회사 10층에서 엘리베이터를 기다리는 일상의 스트레스를 압축 재현.
유저가 호출 버튼을 누를 때마다 다른 방식으로 엘베를 놓치며 분노가 쌓이고,
분노 100 도달 시 엘베를 직접 부수는 **SMASH 모드**로 전환해 카타르시스를 제공.

## 화면 레이아웃

```
┌──────────────────────────────────────────────┐
│  [분노 ██████░░░░ 50/100]              [10F]  │  ← 분노 게이지(중앙) / 현재 층 표시판(우상단)
│                                              │
│  ┌────────┐  ┌────────┐  ┌────────┐          │
│  │  [B2]  │  │  [9F]  │  │  [1F]  │          │  ← 엘베 3대 (LED 층 표시 + 닫힌 철문)
│  │        │  │        │  │        │          │
│  │  문    │  │  문    │  │  문    │          │
│  └────────┘  └────────┘  └────────┘          │
│                                              │
│        "엘레베이터를 부르세요"                │  ← 상태 텍스트
│     ◀◀◀  [  호출  ]  ▶▶▶                    │  ← 호출 버튼 (긴급도 강조 애니메이션)
└──────────────────────────────────────────────┘
```

### 엘베 유닛 구성
- LED 층 표시 패널 (상단, amber 색상 → 층 변경 시 깜빡임 애니메이션)
- 철문 (좌/우 슬라이딩, 이벤트 시 열림/닫힘)
- 문 뒤 군중 (z-index 1, 문이 z-index 2로 가림)
- 상태 배지 (고장/점검중 — 문 위에 팝 애니메이션)

### 분노 게이지 색상 단계

| 범위 | 색상 |
|------|------|
| 0 ~ 29 | 회색 |
| 30 ~ 59 | 코랄 |
| 60 ~ 89 | 레드 |
| 90 ~ 100 | 강렬한 레드 + 게이지 바 흔들림 |

## 핵심 로직

### 상태 머신

```js
state.phase: 'waiting_click' | 'event_playing' | 'smash' | 'ended'
state.eventIndex: 0 ~ 5  (6개 이벤트)
state.rage: 0 ~ 100
```

### 진행 규칙
- 전체 흐름은 **스크립트된 6개 이벤트** 고정 순서 진행 (랜덤화 금지)
- 각 이벤트는 **유저가 호출 버튼을 눌러야만** 시작 (자동 진행 없음)
- 이벤트 중 애니메이션이 모두 완료된 후 분노 게이지 카운트업 → 다음 호출 대기 상태로 복귀

## 이벤트 시나리오 (6개 고정, 합계 분노 100)

| # | 이벤트 설명 | 분노 | 엘베 | 시작 층 |
|---|------------|------|------|--------|
| 1 | 가장 먼 엘베가 잡힘 → 도착했지만 만원 → 엘베 다시 내려감 | +18 | A | B2 |
| 2 | 다른 엘베 호출 → 9층에서 5층으로 내려갔다가 올라오던 중 고장 | +14 | B | 9F |
| 3 | 또 다른 엘베가 9층까지 왔는데 갑자기 B2에 있는 엘베로 배정이 바뀜 | +16 | C→A | C:1F / A:B2 |
| 4 | 배정된 엘베가 B2에서 올라옴 → 도착했지만 동료들이 우르르 타버림 → 1층으로 내려감 | +18 | A | B2 |
| 5 | 남은 엘베 재호출 → 9층에서 10층 오던 중 점검중으로 멈춤 | +14 | C | 9F |
| 6 | 엘베 하나밖에 안 남아서 재호출 → 3층에서 올라옴 → 또 만원 → 분노 100 | +20 | A | 3F |

> **합계: 18+14+16+18+14+20 = 100**
> 이벤트 6 종료 즉시 SMASH 모드 전환

### 이벤트별 연출 상세

**이벤트 1** — 만원
1. A 엘베 강조, LED B2 표시
2. B2 → 10F 층 표시 카운트업 (260ms/층)
3. 군중(crowd-a) 표시 → 문 열림 → "만원이에요!!" → 문 닫힘
4. 군중 숨김, LED 10→B2 배경 하강 연출 후 이벤트 종료

**이벤트 2** — 고장
1. B 엘베 강조, LED 9F
2. 9F → 5F 하강 (310ms/층) → "어? 왜 내려가지...?"
3. 5F → 8F 재상승 (290ms/층) → "다시 올라오네!"
4. LED 깜빡임 6회 → 고장 배지 팝 → "고장으로 운행이 취소됐습니다!"

**이벤트 3** — 배정 변경
1. C 엘베 강조, LED 1F
2. 1F → 9F 상승 (300ms/층) → "9층까지 왔는데...?!"
3. 선택 엘베가 A로 전환, LED B2 표시 → "갑자기 다른 엘베로 배정이 바뀌었습니다!!!"

**이벤트 4** — 동료 새치기
1. A 엘베 강조 (B2에서 출발)
2. B2 → 10F 상승 (260ms/층) → "이번엔 탈 수 있어!!!"
3. 문 열림 + 동료 5명(crowd-b) 오른쪽에서 스태거 슬라이딩 입장 → "동료들이 우르르 타버렸습니다!!"
4. 문 닫힘, 군중 숨김, LED 10→1F 배경 하강 연출

**이벤트 5** — 점검중
1. C 엘베 강조 (9F에서 출발)
2. 9F → 10F 상승 (500ms/층)
3. 점검중 배지 팝 → "점검 중으로 갑자기 멈춰버렸습니다!"

**이벤트 6** — 또 만원 (피날레)
1. A 엘베 강조, LED 3F
2. 3F → 10F 상승 (280ms/층) → "이번엔 진짜 탈 수 있겠지...!!"
3. 군중(crowd-a) 표시 → 문 열림 → "또 만원입니다!!!" → 문 닫힘
4. 분노 100 → SMASH 모드 전환

## 유저 입력 강조 로직 (긴급도 시스템)

버튼이 활성화된 후 누르지 않을 경우 시간에 따라 단계적 강화:

| 경과 시간 | 강조 방식 |
|----------|----------|
| 0 ~ 2초 | 은은한 scale 펄스 (pulse-1) |
| 2 ~ 4초 | 빠른 펄스 + 글로우 + 화살표 속도 증가 (pulse-2) |
| 4 ~ 6초 | 버튼 커짐 + 화면 흔들림 + "빨리 눌러요!" 문구 (pulse-3) |
| 6초 이상 | 최대 강조 + beep 경고음 반복 (pulse-4) |

- 절대 자동 진행 없음
- beep: Web Audio API square wave 880Hz 합성 (외부 오디오 파일 없음)

## SMASH 모드

분노 100 도달 후 활성화.

- "호출" 버튼 → **"부수기"** 버튼으로 교체, 배경 빨간색으로 전환
- 엘베 3대 문 영역 직접 클릭/탭 가능
- 클릭 시: 파편 파티클 (CSS custom property `--fx`, `--fy`, `--fr` 랜덤) + 화면 흔들림 + 노이즈 버스트 타격음

---

## 사람(SVG) 구조

군중과 동료는 인라인 SVG로 표현. 각 `.person` 요소에 CSS 변수로 색상 지정:

```html
<div class="person" style="height:60%;--skin:#E8B898;--hair:#3A2010;--shirt:#3A5A90;--pants:#1A1A2E">
  <svg viewBox="0 0 40 100" preserveAspectRatio="xMidYMax meet">
    <!-- 머리카락, 얼굴, 목, 상의, 팔, 손, 다리, 신발 -->
  </svg>
</div>
```

| 군중 ID | 위치 | 용도 |
|---------|------|------|
| `#crowd-a` | `#frame-a` | 이벤트 1·6 만원 (정적 표시) |
| `#crowd-b` | `#frame-a` | 이벤트 4 동료 탑승 (슬라이딩 입장 애니메이션) |

## 기술 구현 참고

- **층 표시 애니메이션:** `animateFloor(id, from, to, intervalMs, cb)` — 층마다 stepInterval 딜레이, LED 깜빡임 CSS 클래스 재시작
- **분노 게이지:** `requestAnimationFrame` ease-out cubic 보간, 색상 lv1~lv4 자동 전환
- **문 열림/닫힘:** CSS class `door-open` → `.door-l { translateX(-100%) }`, `.door-r { translateX(100%) }`, `.door-seam { opacity:0 }`
- **동료 입장:** CSS `@keyframes personEnter` + `:nth-child` 딜레이 스태거 (0 ~ 0.56s)
- **사운드:** Web Audio API — beep(square wave), 타격음(noise burst)
